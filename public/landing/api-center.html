SPEAXA Hostinger VPS Deployment Guide

1. Core System Prerequisites

This guide assumes you are starting with a fresh **Ubuntu 22.04 LTS VPS** instance hosted on Hostinger. Log in to your VPS server using SSH:

# Login via terminal (replace your-vps-ip with your actual Hostinger VPS IP)
ssh root@your-vps-ip

2. Install Packages (NodeJS, PM2, Git, PostgreSQL)

Run the following script commands to update your libraries, fetch NodeSource NodeJS 20 repositories, install PostgreSQL database engines, and install the PM2 process manager daemon:

# Update package metadata logs
sudo apt update && sudo apt upgrade -y

# Fetch and configure NodeSource repository for NodeJS v20
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs git build-essential

# Install PM2 Process Manager globally
sudo npm install --global pm2

# Install PostgreSQL Database server
sudo apt install postgresql postgresql-contrib -y

3. Setup PostgreSQL Database Schema

Login as a DB superuser, create the database instance, create a database role, grant permissions, and construct the schemas:

# Login to postgres interactive prompt
sudo -i -u postgres psql

# Run SQL commands inside the postgres CLI:
CREATE DATABASE speaxa;
CREATE USER sahil WITH PASSWORD 'your_secure_db_pass';
GRANT ALL PRIVILEGES ON DATABASE speaxa TO sahil;
\q

4. Clone Source Code & Setup Environment

Clone the repository files to `/var/www/speaxa` and configure your production `.env` variables:

# Create folder directory and clone repository
sudo mkdir /var/www && cd /var/www
git clone <your-repository-url> speaxa
cd speaxa

# Install app node dependencies
npm install --omit=dev

# Create production environment variables configuration
nano .env

Paste the following contents into the Nano text editor configuration block (substitute your real keys):

NODE_ENV=production
PORT=5001
DATABASE_URL=postgresql://sahil:your_secure_db_pass@localhost:5432/speaxa
JWT_SECRET=your_super_long_cryptographically_secure_secret_key_here
JWT_EXPIRES_IN=7d

5. Base URL Configuration (Important)

By default, the SPEAXA portals use **relative base URLs** (`const API = '/api';` defined at the top of client files). This is a best practice because **relative paths resolve automatically** to the host domain they are served from. However, if you are splitting client assets onto a separate CDN and hosting the API server on a subdomain (like `api.speaxa.in` or `api.speaxa.com`), you must change the base URLs in the frontend files:

// Locate and edit this variable at the top of client JS files:
const API = 'https://api.speaxa.in/api'; // Point to absolute VPS domain

Verify and edit the constant definition in the following files:

  1. [student.js line 2](file:///Users/sahilkhan/FlutterDev/speaxa_web/public/student/student.js#L2): Change `const API = '/api';`
  2. [teacher.js line 2](file:///Users/sahilkhan/FlutterDev/speaxa_web/public/teacher/teacher.js#L2): Change `const API = '/api';`
  3. [admin.js line 2](file:///Users/sahilkhan/FlutterDev/speaxa_web/public/admin/admin.js#L2): Change `const API = '/api';`
  4. [parent/index.html (styles/JS script block)](file:///Users/sahilkhan/FlutterDev/speaxa_web/public/parent/index.html): Change parent login API target.

6. Nginx Web Server Reverse Proxy Setup

Install and configure Nginx as a reverse proxy. This routes incoming requests on port 80/443 (HTTP/HTTPS) dynamically to the backend Node process on port 5001:

# Install Nginx
sudo apt install nginx -y

# Edit Nginx configuration block
sudo nano /etc/nginx/sites-available/default

Replace the file content with this layout block (substitute your real domain names):

server {
    listen 80;
    server_name speaxa.in www.speaxa.in speaxa.com www.speaxa.com;

    # Route APIs to port 5001
    location /api {
        proxy_pass http://localhost:5001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # Route static frontend portal files
    location / {
        root /var/www/speaxa/public/landing;
        try_files $uri $uri/ =404;
    }
}
# Test Nginx syntax and reload daemon
sudo nginx -t && sudo systemctl reload nginx

7. Enable SSL encryption (HTTPS)

Use Let's Encrypt Certbot to request and configure SSL security certificates on Hostinger VPS:

# Install Certbot libraries
sudo apt install certbot python3-certbot-nginx -y

// Generate certificates (follow automated logs prompt inputs)
sudo certbot --nginx -d speaxa.in -d www.speaxa.in -d speaxa.com -d www.speaxa.com

8. Launch the NodeJS Server with PM2

PM2 runs the server in the background and restarts it automatically if the process crashes or the VPS reboots:

# Start the app server under PM2 management
pm2 start src/app.js --name speaxa-backend

# Persist PM2 across reboots
pm2 startup
# (Copy and run the systemctl command printed on the screen)
pm2 save
